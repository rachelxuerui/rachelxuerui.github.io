Mapping Historical New York
Design exploration and proposal



Overview

I worked with Mapping Historical New York in Fall 2022. The following is a proposed design solution/exploration maximizing the amount of space for the map and streamlining the process for users to select different options to explore the map.

Current design

Current homepage design, screengrab from  mappinghny.com

In this design, there is a large header that takes up a significant portion of the screen containing the name of the project as well as a few options for the user to explore the map.


Problem

How can we keep the full functionality of the existing tools and tabs in this header (Get Started, Stories, About, Process) while minimizing clutter and maximizing the amount of space available for the map?


Solution

Splash page

By designing a splash page as the first page a user encounters upon opening the website, the user is introduced to the name of the project, Mapping Historical New York, thus eliminating the need for the project title in the same page as the map.

In addition, a new user who is new to this project might be overwhelmed at first by the map page with its various options. This splash page introduces a new user to the project, thus orienting them a little before they reach the map page.


Splash page designs, Figma mockup


OPTIONS button and MHNY logo

Now that the splash page introduces the name of the project to the user, I replaced the original, longer logo with an smaller existing MHNY logo, thus reducing space and clutter.

I also consolidated the options Get Started, Stories, About, Process into one drop down menu, labeled Options. I shrunk down the search bar and put this button next to it, effectively eliminating the header and creating more space to view more of the map.
